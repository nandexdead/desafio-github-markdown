Sobre a Formação


Não é possível falar sobre esse projeto sem antes falar sobre suas origens. Iniciei o curso "Formação GitHub Certification" com o intuito de aumentar meus conhecimentos sobre versionamento de código, aprimorando assim, minha carreira como futura desenvolvedora FullStack. Durante o curso, até o presente momento, aprendi sobre os comandos Git, Autenticações, Markdown, colaboração básica no GitHub e diversos outros parâmetros, através dos valiosos ensinamentos da professora "Aline Antunes", analyst tech education na plataforma de capacitação "DIO (Digital Innovation One)". A formação conta com 14 atividades (9 pequenos cursos separados em capítulos, 1 ítem que contém cinco desafios de código, 3 desafios de projeto, e ao final uma seção especial dedicada à conclusão do curso) totalmente voltadas para iniciantes nesse mundo vasto do desenvolvimento e colaboração, separadas em 3 tópicos principais, totalizando assim, 10 horas:


  • Introdução aos conceitos de Git e GitHub

  • Produtos do GitHub

  • Administração do GitHub

Para conferir mais informações sobre o curso, acesse: Formação GitHub Certification, e caso se interesse, matricule-se! Você irá se deparar com uma formação aprazível, inovadora, com conteúdo profundamente relevante e estrutura didática impecável! Além de uma abordagem intuitiva que possibilita a facilidade de absorção do conteúdo a ser estudado!
Ainda tenho um longo caminho pela frente, mas enfim chegou o momento de colocar todo esse conhecimento adquirido!


Desafio de Código - Explorando Colaboração e Markdown


Ao fim do cumprimento da primeira etapa do curso, me deparei com o desafio de código "Explorando Colaboração e Markdown", onde o objetivo é criar um projeto de nome "Desafio GitHub Markdown", e atender a uma série de requisitos, como:


  • Criar uma conta no GitHub (um pouco óbvio )

  • Criar um repositório

  • Trabalhar a colaboração no repositório criado

  • Formatação do README.md com Markdown

  • Além de alguns desafios extra


  • Mais informações? leia o README.md desse projeto!

Para tais finalizades, convidei o analista de sistemas Fernando Dias para desenvolver e colaborar neste projeto comigo, e assim, concluir com sucesso os objetivos do desafio!
Com a intenção de não tirar o brilho dos propósitos principais do projeto, que são a colaboração e o markdown, nos inspiramos na criação de um simples site informativo que conta com o detalhamento sobre os processos, história do curso, intenções da experiência prática, e claro, sobre os desenvolvedores/colaboradores!
